# PAAKURI 작업 인수인계

## 현재 결과

Godot 4.7.1 Mono(C#, .NET 8)로 멀티플레이를 제외한 오프라인 전투 프로토타입을 구현했다. 시작 화면에서 게임을 실행하면 한 명의 사용자가 Player 1을 조작하고 Player 2 전투 봇과 3판 2선승 경기를 진행한다. 혼자서도 이동, 탄도, 피해, 패링, 탄약, 라운드 흐름을 반복 테스트할 수 있다.

새 이미지/음원/폰트 자산은 만들지 않았다. 외부 이미지 참조는 기존 `resources/map1.png`와 `resources/marketing.png`뿐이다. 플레이어, 총, 총알, 패링, HUD 및 이펙트는 Godot 장면의 기본 도형과 노드로 표시한다.

## 구현한 기능

### 게임 흐름

- 시작 화면과 게임 시작 버튼
- 3초 카운트다운, 60초 라운드, 3판 2선승
- 라운드마다 체력, 위치, 속도, 탄약, 재장전, 패링 상태, 총알 초기화
- 체력 0 또는 화면 아래 낙사 시 라운드 종료
- 제한 시간 종료 시 체력이 높은 플레이어 승리
- 제한 시간에 체력이 같거나 동시 사망하면 무승부로 처리하고 해당 라운드를 재경기
- 경기 종료 후 `R`로 전체 경기 재시작, `ESC`로 시작 화면 이동

### 플레이어 이동

- A/D 즉시 좌우 이동 및 즉시 정지
- Space 점프, 0.1초 코요테 타임, 0.1초 점프 입력 버퍼
- 공중 추가 점프 금지
- 벽 방향 입력 시 벽 붙기/미끄러지기
- 벽에서 W로 느린 상승, S로 하강, Space로 벽 점프
- 벽 점프 직후 0.15초 수평 입력 잠금
- 공중 S 빠른 낙하
- 지상 S 숙이기 및 저속 이동(120px/s), 충돌 높이 감소

### 전투

- 마우스 방향 조준 및 총 회전
- 중력을 받는 포물선 총알, 연속 충돌 감지를 사용한 빠른 총알 충돌
- 4발 탄창, 0.7초 발사 간격, 소진 시 3초 자동 재장전
- 발사 반대 방향 총기 반동
- 총알 25 피해, 최대 체력 100, 0.15초 피격 무적
- 총알 진행 방향 기반 넉백
- 벽/플랫폼 충돌 시 총알 제거, 파티클과 카메라 흔들림
- 총알 수명 5초 및 라운드 종료 시 일괄 제거
- 패링 준비 0.05초, 활성 0.15초, 후딜레이 0.25초, 쿨타임 0.8초
- 패링 성공 시 소유자 변경, 진행 방향 반전, 속도 15% 증가
- 총알 소유자 충돌 유예로 발사 직후 자해 방지

### 테스트 지원

- Player 2 봇은 거리 유지, 점프/벽 이동, 탄도 보정 조준, 사격, 접근 총알 패링을 수행한다.
- HUD에 양측 체력, 4발 탄약, 재장전 진행률, 패링 쿨타임, 점수, 라운드 시간을 표시한다.
- 화면 하단에 조작법을 항상 표시한다.
- 주요 피격/발사/패링/충돌에 카메라 흔들림과 간단한 도형 이펙트를 적용했다.

## 조작법

| 입력 | 기능 |
| --- | --- |
| A / D | 좌우 이동 |
| W | 벽에 붙었을 때 위로 이동 |
| S | 지상 숙이기 / 공중 빠른 낙하 / 벽 아래 이동 |
| Space | 점프 / 벽 점프 |
| 마우스 | 조준 |
| 마우스 좌클릭 | 사격 |
| 마우스 우클릭 | 패링 |
| R | 현재 경기를 0:0부터 재시작 |
| ESC | 시작 화면으로 이동 |

## 주요 파일

- `Player.cs`: 입력, 이동, 봇, 조준, 사격, 재장전, 피해, 패링
- `Bullet.cs`: 탄도 물리, 소유권, 충돌, 피해, 반사, 수명, 충돌 이펙트
- `GameManager.cs`: 카운트다운, 라운드/경기 상태, 점수, 타임아웃, 낙사, HUD
- `CameraShake.cs`, `DamageOverlay.cs`: 화면 피드백
- `Scene/main.tscn`: 임시 맵 충돌, 두 플레이어, HUD, 카메라
- `Scene/start.tscn`: 시작 화면
- `project.godot`: 입력 맵과 1920x1080 화면 설정

## 검증 결과

2026-07-21 기준 다음 검증을 통과했다.

1. `C:\Program Files\dotnet\dotnet.exe build Master_JS.csproj --nologo`
   - 경고 0개, 오류 0개
2. Godot 4.7.1 Mono 헤드리스 에디터 import/scene parse
   - 스크립트 클래스와 장면 로딩 성공
3. `main.tscn` 헤드리스 런타임 스모크 테스트(420프레임)
   - 런타임 예외 없이 종료
4. 현재 작업공간을 경로 지정해 일반 Godot 창에서 렌더 확인
   - HUD, 임시 맵, 두 캐릭터, 탄도 총알 표시 확인
   - C# Export 직렬화 키 문제를 발견해 Player 2 봇/색상 설정을 PascalCase로 수정

헤드리스 dummy renderer에서 Movie Maker로 화면 캡처를 시도하면 Godot 엔진 자체가 null texture 오류로 종료됐다. 일반 헤드리스 게임 실행과 C# 빌드는 정상이며, 렌더 화면 확인은 일반 Godot 창에서 수행해야 한다.

## 현재 한계와 결정 사항

- 온라인/방 코드/호스트 권한/동기화는 의도적으로 구현하지 않았다.
- 제공된 임시 맵 한 종만 사용한다.
- 사운드 자산이 없어 사운드는 구현하지 않았다.
- 애니메이션 자산이 없어 플레이어는 장면 도형으로 표시한다.
- 숙인 상태에서는 완전 정지가 아니라 120px/s로 이동하도록 결정했다.
- 수동 재장전은 없고 탄창 소진 시에만 자동 재장전한다.
- 패링은 조준 방향 재발사가 아니라 기존 속도의 정반대 반사다.
- 같은 맵 중복 문제는 맵이 하나뿐이므로 아직 해당되지 않는다.
- 봇은 기능 테스트용이며 정교한 길찾기나 공정한 상대 AI가 아니다.
