# PAAKURI 게임 기획서

## 1. 게임 개요

### 1.1 한 줄 소개

포물선으로 날아가는 총알과 패링을 활용해 상대를 쓰러뜨리는 1대1 온라인 2D 플랫포머 슈팅 게임.

### 1.2 게임 콘셉트

PAAKURI는 **ROUNDS와 유사한 2D 플랫포머 대전 구조**를 기반으로 하되, 카드 선택과 증강 시스템을 제거하고 이동, 사격, 패링에 집중한 게임이다.

모든 플레이어는 같은 캐릭터와 같은 무기를 사용한다. 승패는 능력치나 빌드 차이가 아니라 다음 요소로 결정된다.

* 조준 실력
* 이동 판단
* 패링 타이밍
* 탄약 관리
* 맵과 지형 활용
* 총기 반동 활용

### 1.3 기본 정보

| 항목          | 내용                 |
| ----------- | ------------------ |
| 장르          | 1대1 온라인 2D 플랫포머 슈팅 |
| 플레이 인원      | 2명                 |
| 경기 방식       | 3판 2선승             |
| 목표 프레임      | 60FPS              |
| 라운드 시간      | 최대 60초             |
| 전체 경기 시간    | 약 2~5분             |
| 맵 개수        | 5종                 |
| 무기 종류       | 1종                 |
| 증강 시스템      | 없음                 |
| 파괴 가능한 오브젝트 | 없음                 |
| 핵심 요소       | 이동, 포물선 사격, 패링, 넉백 |

## 2. 핵심 재미

### 2.1 포물선 사격

총알은 직선으로 이동하지 않고 중력의 영향을 받아 포물선으로 날아간다.

플레이어는 상대와의 거리, 높이 차이, 이동 방향을 고려하여 조준해야 한다. 벽이나 플랫폼 뒤에 있는 상대에게 총알을 넘겨 쏘는 플레이도 가능하다.

### 2.2 패링 심리전

플레이어는 상대가 발사한 총알을 패링하여 되돌려 보낼 수 있다.

공격자는 상대의 패링 타이밍을 예상하며 발사해야 하고, 방어자는 총알이 접근하는 순간에 맞춰 패링해야 한다.

패링은 성공 시 강력하지만 실패하면 후딜레이와 쿨타임이 발생하므로 무작정 사용할 수 없다.

### 2.3 이동과 위치 선정

플레이어는 다음 이동 기술을 활용한다.

* 좌우 이동
* 점프
* 벽 붙기
* 벽 오르기
* 벽 미끄러지기
* 벽 점프
* 빠른 낙하

총기 반동과 피격 넉백으로 캐릭터의 위치가 변경되므로, 단순히 상대의 체력을 줄이는 것뿐 아니라 상대를 위험한 위치로 밀어내는 것도 중요하다.

## 3. 게임 진행 방식

### 3.1 매치 시작

1. 플레이어가 방을 생성하거나 기존 방에 참가한다.
2. 두 플레이어가 모두 입장한다.
3. 두 플레이어가 준비 상태가 된다.
4. 5개의 맵 중 하나가 무작위로 선택된다.
5. 두 플레이어가 맵의 반대편 시작 지점에 생성된다.
6. 3초 카운트다운 후 라운드가 시작된다.

### 3.2 라운드 진행

라운드가 시작되면 플레이어는 맵을 이동하며 상대를 공격한다.

상대방의 체력을 0으로 만들거나 낙사시키면 라운드에서 승리한다.

### 3.3 라운드 종료

한 명의 플레이어가 사망하면 라운드가 종료된다.

라운드 종료 후 다음 요소를 초기화한다.

* 플레이어 체력
* 플레이어 위치
* 플레이어 속도
* 남은 탄약
* 재장전 상태
* 맵에 남아 있는 총알
* 패링 상태와 쿨타임
* 피격 무적 상태

다음 라운드에서는 5개의 맵 중 하나를 다시 무작위로 선택한다.

### 3.4 경기 종료

먼저 2승을 달성한 플레이어가 최종 경기에서 승리한다.

| 항목           | 설정   |
| ------------ | ---- |
| 최대 라운드       | 3라운드 |
| 필요 승리 수      | 2승   |
| 라운드 제한 시간    | 60초  |
| 라운드 사이 대기 시간 | 3초   |
| 시작 카운트다운     | 3초   |

## 4. 승리 및 패배 조건

### 4.1 라운드 승리 조건

다음 조건 중 하나를 만족하면 라운드에서 승리한다.

* 상대 플레이어의 체력을 0으로 만든다.
* 상대 플레이어를 맵 아래의 사망 구역으로 떨어뜨린다.

### 4.2 라운드 패배 조건

다음 조건 중 하나를 만족하면 라운드에서 패배한다.

* 자신의 체력이 0 이하가 된다.
* 맵 아래의 사망 구역에 진입한다.

### 4.3 경기 승리 조건

3판 중 먼저 2승을 달성한 플레이어가 경기에서 승리한다.

## 5. 플레이어 조작

| 입력      | 기능                |
| ------- | ----------------- |
| A       | 왼쪽 이동             |
| D       | 오른쪽 이동            |
| W       | 벽에 붙은 상태에서 위로 이동  |
| S       | 지상에서 숙이기          |
| S       | 공중에서 빠른 낙하        |
| S       | 벽에 붙은 상태에서 아래로 이동 |
| Space   | 점프 또는 벽 점프        |
| 마우스 이동  | 조준 방향 설정          |
| 마우스 좌클릭 | 총알 발사             |
| 마우스 우클릭 | 패링                |
| ESC     | 메뉴 열기             |

W와 S는 일반적인 상하 이동 입력이 아니다. 플레이어의 현재 상태에 따라 벽 이동, 숙이기, 빠른 낙하 기능으로 변경된다.

## 6. 플레이어 이동 시스템

### 6.1 좌우 이동

플레이어는 A와 D를 이용해 좌우로 이동한다.

이동에는 별도의 가속과 감속을 적용하지 않는다.

* 이동 키를 누르면 즉시 최대 이동 속도로 이동한다.
* 이동 키에서 손을 떼면 즉시 정지한다.
* 양쪽 플레이어는 동일한 이동 속도를 사용한다.

빠르고 단순한 조작감을 목표로 한다.

### 6.2 점프

Space를 누르면 점프한다.

* 지상에서는 일반 점프를 사용한다.
* 공중에서는 추가 점프할 수 없다.
* 벽에 붙은 상태에서는 벽 점프로 변경된다.

| 기능       | 설정          |
| -------- | ----------- |
| 코요테 타임   | 0.1초        |
| 점프 입력 버퍼 | 0.1초        |
| 최대 점프 높이 | 캐릭터 키의 약 2배 |
| 공중 추가 점프 | 불가능         |

### 6.3 벽 붙기

플레이어가 공중에서 벽 방향으로 이동하면 벽에 붙는다.

벽에 붙은 상태에서는 일반적인 자유 낙하보다 느리게 내려간다.

| 행동       | 조건                  |
| -------- | ------------------- |
| 벽 붙기     | 공중에서 벽 방향 입력        |
| 벽 미끄러지기  | 벽에 붙은 상태에서 위쪽 입력 없음 |
| 벽 위로 이동  | 벽에 붙은 상태에서 W 입력     |
| 벽 아래로 이동 | 벽에 붙은 상태에서 S 입력     |
| 벽 점프     | 벽에 붙은 상태에서 Space 입력 |

벽에서 무한히 안전하게 버티는 상황을 방지하기 위해 벽 위로 올라가는 속도는 느리게 설정한다.

필요한 경우 벽에 붙어 있을 수 있는 최대 시간을 추가한다.

### 6.4 벽 점프

벽에 붙은 상태에서 Space를 누르면 벽 반대 방향으로 점프한다.

벽 점프 직후에는 짧은 시간 동안 반대 방향으로 힘을 적용하여 플레이어가 다시 같은 벽에 즉시 붙지 않도록 한다.

### 6.5 빠른 낙하

공중에서 S를 누르면 평소보다 빠르게 아래로 떨어진다.

빠른 낙하는 다음 상황에서 활용할 수 있다.

* 상대의 포물선 조준 회피
* 낮은 플랫폼으로 빠르게 이동
* 공중 체류 시간 감소
* 낙하 타이밍 변경

### 6.6 숙이기

지상에서 S를 누르면 캐릭터가 숙인다.

숙인 상태에서는 캐릭터의 충돌 판정 높이가 감소하여 일부 총알을 피할 수 있다.

숙이는 동안에는 좌우 이동 속도를 제한하거나 이동할 수 없도록 설정한다.

## 7. 조준 시스템

플레이어 캐릭터는 마우스 커서 방향으로 총을 조준한다.

조준 방향은 다음 과정으로 계산한다.

1. 마우스의 화면 좌표를 월드 좌표로 변환한다.
2. 플레이어 또는 총구 위치에서 마우스 월드 좌표까지의 방향을 계산한다.
3. 계산된 방향으로 총의 회전을 변경한다.
4. 조준 방향에 따라 캐릭터 스프라이트를 좌우 반전한다.

상대 플레이어도 현재 총의 방향을 볼 수 있다. 이를 통해 상대방이 어느 방향으로 공격하려는지 어느 정도 예상할 수 있다.

다음 기능은 제공하지 않는다.

* 별도의 조준선
* 포물선 예상 경로
* 착탄 지점 표시
* 자동 조준

## 8. 사격 시스템

### 8.1 기본 사격

마우스 좌클릭을 누르면 현재 조준 방향으로 총알을 발사한다.

총은 한 종류만 존재하며, 두 플레이어 모두 같은 성능을 사용한다.

| 항목       | 초기 설정 |
| -------- | ----- |
| 최대 장탄 수  | 4발    |
| 발사 간격    | 0.7초  |
| 재장전 시간   | 3초    |
| 총알 피해량   | 25    |
| 처치 필요 횟수 | 기본 4회 |
| 자동 재장전   | 사용    |

### 8.2 탄약과 재장전

총에는 최대 4발의 탄약이 들어간다.

총알을 모두 사용하면 자동으로 재장전을 시작한다.

재장전 중에는 사격할 수 없으며, 재장전이 완료되면 탄약이 4발로 회복된다.

패링과 이동은 재장전 중에도 사용할 수 있다.

### 8.3 발사 제한

다음 상태에서는 총알을 발사할 수 없다.

* 발사 쿨타임이 남아 있는 상태
* 재장전 중인 상태
* 패링 판정이 활성화된 상태
* 사망한 상태
* 카운트다운 또는 라운드 종료 상태

### 8.4 총기 반동

총알을 발사하면 플레이어는 발사 방향의 반대쪽으로 약하게 밀려난다.

반동은 지상과 공중 모두 적용된다.

공중에서는 반동을 이용해 다음과 같은 이동이 가능하다.

* 낙하 방향 조정
* 벽 쪽으로 접근
* 상대와 거리 벌리기
* 낙사 직전 위치 보정

반동은 이동을 보조하는 정도로 적용한다. 연속 사격만으로 공중에 계속 머물 수 있을 정도로 강하게 설정하지 않는다.

## 9. 포물선 총알 시스템

### 9.1 총알 이동

총알은 생성된 순간부터 중력의 영향을 받는다.

총알의 이동은 다음 요소로 결정된다.

* 발사 위치
* 발사 방향
* 초기 속도
* 중력값
* 현재 속도
* 생존 시간

가까운 상대에게는 직접 조준할 수 있지만, 거리가 멀수록 상대보다 높은 위치를 조준해야 한다.

### 9.2 총알 특징

* 모든 총알은 동일한 피해량을 가진다.
* 총알 속도에 따라 피해량이 변하지 않는다.
* 다른 총알과 충돌하지 않는다.
* 벽이나 바닥에 충돌하면 제거된다.
* 일정 시간이 지나면 자동으로 제거된다.
* 패링되면 소유자와 진행 방향이 변경된다.

### 9.3 총알 충돌

| 충돌 대상    | 결과                  |
| -------- | ------------------- |
| 상대 플레이어  | 피해와 넉백 적용 후 총알 제거   |
| 발사한 플레이어 | 발사 직후 일정 시간 충돌하지 않음 |
| 벽        | 충돌 효과 출력 후 제거       |
| 바닥 및 플랫폼 | 충돌 효과 출력 후 제거       |
| 패링 범위    | 총알 소유자와 방향 변경       |
| 다른 총알    | 서로 통과               |
| 사망 구역    | 즉시 제거               |

### 9.4 발사자 충돌 방지

총알이 생성된 직후 발사한 플레이어에게 충돌하는 문제를 방지해야 한다.

다음 방법 중 하나를 사용한다.

* 생성 후 짧은 시간 동안 발사자와 충돌하지 않음
* 총알이 발사자의 충돌 범위를 벗어날 때까지 충돌하지 않음
* 총알 소유자 ID를 이용해 충돌을 무시함

## 10. 패링 시스템

### 10.1 기본 패링

마우스 우클릭을 누르면 캐릭터가 바라보는 방향에 짧은 패링 판정이 생성된다.

패링 범위 안에 상대방의 총알이 들어오면 해당 총알을 반사한다.

```text
플레이어 A가 총알 발사
→ 플레이어 B가 패링
→ 총알 진행 방향 변경
→ 총알 소유자가 플레이어 B로 변경
→ 플레이어 A가 맞으면 피해 적용
```

### 10.2 패링 단계

| 단계       | 시간    |
| -------- | ----- |
| 패링 준비 시간 | 0.05초 |
| 패링 판정 시간 | 0.15초 |
| 패링 후딜레이  | 0.25초 |
| 패링 쿨타임   | 0.8초  |

우클릭을 계속 누르고 있어도 패링 상태가 유지되지 않는다.

패링 버튼을 누른 순간부터 정해진 시간 동안만 판정이 활성화된다.

### 10.3 패링 성공

패링에 성공하면 다음 효과를 적용한다.

* 총알 소유자 변경
* 총알 진행 방향 변경
* 총알 속도 15% 증가
* 패링 이펙트 출력
* 패링 성공 효과음 출력
* 작은 카메라 흔들림
* 캐릭터에 짧은 반짝임 표시

패링된 총알의 피해량은 증가하지 않는다.

### 10.4 패링 방향

패링된 총알은 단순히 기존 방향의 정반대로 되돌리는 방식으로 처리한다.

프로토타입에서는 패링한 플레이어의 조준 방향으로 총알을 새롭게 발사하는 기능은 적용하지 않는다.

### 10.5 패링 실패

패링에 실패해도 추가 피해를 받지는 않는다.

다만 패링 후딜레이와 쿨타임은 그대로 적용된다.

* 패링 중에도 이동 가능
* 패링 판정 중에는 사격 불가능
* 패링 종료 후 사격 가능
* 쿨타임 중에는 다시 패링 불가능

---

## 11. 체력과 피해 시스템

### 11.1 체력

모든 플레이어의 최대 체력은 100이다.

라운드가 시작될 때 체력을 최대치로 초기화한다.

| 공격     | 피해량   |
| ------ | ----- |
| 일반 총알  | 25    |
| 패링된 총알 | 25    |
| 낙사     | 즉시 사망 |

기본적으로 총알에 4번 맞으면 사망한다.

### 11.2 피해 처리

플레이어가 유효한 총알에 맞으면 다음 순서로 처리한다.

1. 총알 소유자가 피격 대상과 다른지 확인한다.
2. 피격 무적 상태인지 확인한다.
3. 체력을 감소시킨다.
4. 총알 방향을 기준으로 넉백을 적용한다.
5. 피격 이펙트와 효과음을 재생한다.
6. 체력이 0 이하인지 확인한다.
7. 사망 조건을 만족하면 라운드를 종료한다.

### 11.3 넉백

총알에 맞으면 총알이 날아온 방향으로 밀려난다.

넉백은 다음 역할을 담당한다.

* 피격 타격감 제공
* 상대 위치 변경
* 낙사 가능성 생성
* 연속 공격 방지
* 유리한 위치에서 상대 밀어내기

넉백의 힘은 현재 체력과 관계없이 일정하게 적용한다.

### 11.4 피격 무적 시간

짧은 시간 동안 여러 총알이 동시에 충돌하는 문제를 방지하기 위해 피격 후 약 0.15초의 무적 시간을 적용한다.

무적 시간 동안에는 추가 피해를 받지 않지만 다음 행동은 가능하다.

* 이동
* 점프
* 사격
* 패링

## 12. 사망과 낙사

### 12.1 체력 사망

체력이 0 이하가 되면 해당 플레이어가 사망한다.

사망한 순간부터 다음 입력을 비활성화한다.

* 이동
* 점프
* 사격
* 패링
* 재장전

### 12.2 낙사

플레이어가 맵 아래에 배치된 사망 구역에 진입하면 남은 체력과 관계없이 즉시 사망한다.

낙사는 다음 상황에서 발생할 수 있다.

* 이동 실수
* 벽 점프 실패
* 상대 총알의 넉백
* 총기 반동 사용 실수
* 빠른 낙하 사용 실수

### 12.3 사망 연출

사망 시 다음 연출을 적용한다.

* 강한 카메라 흔들림
* 캐릭터가 마지막 넉백 방향으로 날아감
* 짧은 슬로 모션
* 라운드 승자 표시
* 라운드 점수 갱신

온라인 게임 전체의 시간을 실제로 느리게 하지 않는다.

슬로 모션은 각 클라이언트의 화면, 애니메이션, 카메라 연출에만 적용한다. 호스트의 네트워크 로직과 게임 판정은 정상 속도로 유지한다.

## 13. 타격감 연출

복잡한 시각 효과보다는 짧고 명확한 반응을 사용한다.

| 상황      | 연출                 |
| ------- | ------------------ |
| 총알 발사   | 총구 화염, 약한 반동, 발사음  |
| 총알 벽 충돌 | 작은 파편, 충돌음         |
| 플레이어 피격 | 점멸, 넉백, 카메라 흔들림    |
| 패링 사용   | 짧은 패링 모션, 사용음      |
| 패링 성공   | 원형 파동, 강한 효과음, 반짝임 |
| 플레이어 사망 | 강한 흔들림, 짧은 슬로 모션   |

### 13.1 카메라 흔들림

| 상황      | 세기    |
| ------- | ----- |
| 총알 발사   | 매우 약함 |
| 총알 벽 충돌 | 약함    |
| 일반 피격   | 중간    |
| 패링 성공   | 중간    |
| 사망      | 강함    |

카메라 흔들림은 짧게 적용하여 조준과 이동을 방해하지 않도록 한다.

## 14. 맵 구성

### 14.1 기본 구성

모든 맵은 고정된 벽과 플랫폼으로만 구성한다.

다음 요소는 사용하지 않는다.

* 파괴 가능한 박스
* 움직이는 플랫폼
* 작동형 함정
* 랜덤으로 생성되는 지형
* 플레이어가 이동시킬 수 있는 오브젝트

맵 전체는 한 화면 안에 표시되며, 플레이어를 따라 카메라가 크게 이동하지 않는다.

### 14.2 맵 개수

총 5개의 맵을 제작한다.

각 라운드가 시작될 때 맵을 무작위로 선택한다.

같은 경기에서 동일한 맵이 다시 선택될 수 있는지는 추후 결정한다.

### 14.3 맵 설계 원칙

모든 맵은 다음 조건을 만족해야 한다.

* 좌우 시작 위치가 공평해야 한다.
* 가급적 좌우 대칭 구조로 제작한다.
* 시작 직후 바로 피격되지 않아야 한다.
* 벽 붙기와 벽 점프를 사용할 수 있어야 한다.
* 적어도 두 개 이상의 이동 경로가 있어야 한다.
* 포물선 사격을 활용할 수 있는 공간이 있어야 한다.
* 맵 한쪽에서 반대쪽까지 약 5~8초 안에 이동할 수 있어야 한다.
* 특정 위치에서 무한히 방어할 수 없어야 한다.
* 플레이어가 화면 밖으로 지나치게 멀리 이동할 수 없어야 한다.
* 낙사 위험 지역이 지나치게 한쪽에 유리하지 않아야 한다.

## 15. 멀티플레이 구조

### 15.1 기본 방식

플레이어 한 명이 방을 생성하고, 다른 플레이어가 방 코드를 입력해 참가한다.

프로토타입에서는 한 명의 플레이어가 호스트 역할을 수행하는 호스트 기반 구조를 사용한다.

호스트가 주요 게임 판정을 담당하며, 참가자는 입력과 요청을 호스트에 전달한다.

### 15.2 호스트 권한 방식

주요 게임 결과는 호스트가 결정한다.

클라이언트가 직접 상대방의 체력을 감소시키거나 라운드를 종료할 수 없다.

```text
클라이언트 입력
→ 호스트에 요청 전송
→ 호스트가 요청 검증
→ 호스트가 게임 상태 변경
→ 변경된 결과를 모든 클라이언트에 전달
```

### 15.3 호스트 책임

호스트는 다음 로직을 처리한다.

* 플레이어 입장 및 생성
* 플레이어 준비 상태 관리
* 맵 선택
* 라운드 시작과 종료
* 플레이어 이동 상태 동기화
* 총알 생성
* 총알 이동 및 충돌 판정
* 패링 성공 판정
* 피해 적용
* 체력 관리
* 넉백 적용
* 사망과 낙사 판정
* 라운드 점수 관리
* 최종 승리 판정

### 15.4 클라이언트 책임

클라이언트는 다음 기능을 담당한다.

* 키보드 및 마우스 입력 수집
* 이동 입력 전달
* 조준 방향 전달
* 사격 요청 전달
* 패링 요청 전달
* 애니메이션 재생
* 카메라 흔들림
* 이펙트 출력
* 사운드 출력
* UI 갱신
* 호스트가 전달한 위치와 상태 보간

### 15.5 주요 네트워크 데이터

| 데이터         | 전송 방향       |
| ----------- | ----------- |
| 이동 입력       | 클라이언트 → 호스트 |
| 조준 방향       | 클라이언트 → 호스트 |
