# PAAKURI 다음 개발 작업

마지막 갱신: 2026-07-21
대상 환경: Godot 4.7.1 Mono, .NET 8, Windows
현재 단계: 오프라인 전투 프로토타입 안정화 및 게임 흐름 완성

## 공통 제약

- 이미지, 사운드, 애니메이션 등 에셋은 반드시 `resources/`에 이미 존재하는 파일만 사용한다.
- 현재 사용 가능한 완성 맵은 `map_a.png`, `map_c.png`, `map_d.png`, `map_g.png` 네 개다.
- 플레이어는 애니메이션 없는 임시 캐릭터 상태다. 새 캐릭터/아이콘/맵/사운드 에셋을 만들지 않는다.
- 사용자 작업과 무관한 파일은 되돌리거나 삭제하지 않는다.
- 각 작업 후 `work.md`에 변경 파일, 검증 명령, 결과, 남은 위험을 기록한다.

## P0 — 다음 작업에서 우선 검증·수정

### MAP-001 네 맵 수동 플레이 검증

- [ ] 각 맵을 강제로 한 번씩 선택할 수 있는 개발용 선택 방법을 코드 또는 에디터 export 값으로 제공한다. 배포 기본값은 무작위다.
- [ ] 네 맵에서 양쪽 스폰이 지형 위에 안전하게 착지하며 시작 직후 즉시 피격되지 않는지 확인한다.
- [ ] 알파 윤곽 충돌이 화면의 흰색/회색 지형과 일치하는지 확인한다.

- [ ] 각 맵에서 벽 붙기, 벽 오르기, 벽 점프가 가능한 구간을 확인한다.
- [ ] 맵 좌우 이동 시간이 약 5~8초인지 측정하고 스폰 좌표 또는 이동 수치를 조정한다.

완료 조건: 네 맵별 결과와 수정 좌표를 `work.md`에 기록하고, 각 맵 3라운드 연속 플레이 중 충돌 오류가 없어야 한다.
