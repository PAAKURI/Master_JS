# PAAKURI 작업 인수인계

마지막 갱신: 2026-07-21
환경: Godot 4.7.1 Mono, .NET 8, Windows

## 이번 작업 결과

변경된 `AGENTS.md`와 기존 `task.md`를 기준으로 오프라인 1대1 전투 프로토타입을 통합·안정화했다. 기존 충돌 상태였던 `Player.cs`와 `Scene/main.tscn`은 기능이 더 완성된 오프라인 전투 구현을 기준으로 병합했고, 별도 이미지·사운드·애니메이션 에셋은 만들지 않았다.

### 경기 흐름

- `GameManager`를 `Countdown / Playing / RoundEnd / MatchEnd / Result` 상태 기반으로 변경했다.
- 3초 카운트다운, 60초 라운드, 라운드 사이 3초, 먼저 2승 시 경기 종료 흐름을 유지했다.
- 라운드 종료 즉시 두 플레이어의 입력과 피해 판정을 끄고 남은 총알을 제거한다.
- 같은 물리 프레임의 사망 신호를 deferred 처리로 모은다. 양쪽이 모두 사망하면 무승부로 처리하고 점수를 올리지 않은 채 라운드를 다시 시작한다.
- 시간 초과 시 체력이 높은 쪽이 승리한다. 체력이 같으면 무승부 재경기다.
- R/ESC는 `restart_match`, `open_menu` InputMap 액션의 edge 입력으로 처리한다.
- 경기 종료 결과 패널에 승자, 최종 점수, 재대결, 타이틀, 종료 버튼을 연결했다.

### 맵

- 완성된 기존 리소스 `map_a.png`, `map_c.png`, `map_d.png`, `map_g.png`만 경기 맵으로 사용한다.
- 라운드마다 네 맵 중 하나를 무작위로 선택한다. 현재 프로토타입은 같은 경기 내 중복 선택을 허용한다.
- PNG의 알파 윤곽을 `Bitmap.OpaqueToPolygons`로 변환하여 지형 충돌을 런타임에 구성한다. 별도 충돌 이미지나 신규 에셋은 없다.
- 각 맵별 좌우 대칭 스폰 좌표를 `GameManager.MapDefinition`에 지정했다.

### 플레이어와 전투

- 이동, 점프, 코요테 타임, 입력 버퍼, 벽 붙기/오르기/미끄러지기/점프, 빠른 낙하, 숙이기 구현을 유지했다.
- 포물선 총알, 4발 탄창, 0.7초 발사 간격, 3초 자동 재장전, 반동, 25 피해, 넉백, 0.15초 피격 무적을 유지했다.
- 패링은 준비 0.05초, 활성 0.15초, 후딜 0.25초, 쿨타임 0.8초이며 총알 소유자를 변경하고 진행 방향을 반전해 속도를 15% 높인다.
- 발사자 충돌은 생성 후 0.15초만 제외한다. 패링 뒤에는 새 소유자에게 같은 유예가 적용되고 이전 소유자는 피격 가능하다.
- `CombatEnabled`로 카운트다운과 라운드 종료 중의 추가 피해를 차단한다.
- 숙일 때 충돌 도형의 발 위치가 유지되도록 높이와 위치를 함께 조정한다.
- 각 플레이어 머리 위에 숫자 없는 체력바를 추가했다. 화면 상단 상태 UI에는 체력 숫자를 제거하고 탄약/재장전/패링 상태만 남겼다.

## 변경 파일

- `GameManager.cs`: 경기 상태, 동시 사망, 맵 선택/충돌 생성, 결과 흐름
- `Player.cs`: 병합 충돌 해소, 전투 판정 게이트, 체력바 갱신, 숙이기 발 위치 보정
- `Bullet.cs`: 제거된 총알의 발사자 충돌 유예 타이머가 해제 객체를 참조하지 않도록 수명 검사
- `GameOverPanel.cs`: 승자/점수 표시와 재대결·타이틀·종료 버튼
- `Scene/main.tscn`: 네 맵용 동적 Arena, 머리 위 체력바, 결과 패널, HUD 통합
- `project.godot`: `restart_match`, `open_menu` 입력 액션
- `task.md`: 다음 작업 우선순위와 검증 기준 갱신

## 알려진 한계와 다음 에이전트 주의사항

- 실제 렌더러에서 네 맵의 충돌 윤곽, 스폰 안전성, 벽 점프 가능 구간을 각각 수동 플레이로 확인해야 한다. 헤드리스 스모크 테스트는 조작감과 픽셀 경계를 검증하지 않는다.
- 낮은 천장 아래에서 숙이기를 해제할 때 일어설 공간을 검사하는 기능은 아직 없다. 발 위치 보정만 완료됐다.
- 머리 위 체력바는 추가됐지만 총알과 패링 범위에 가려지는지 실제 화면 확인이 필요하다.
- 맵 선택 중복 허용, 벽 붙기 최대 시간, 숙인 채 이동 속도, 수동 재장전은 아직 최종 기획 확정이 아니다.
- 사운드와 애니메이션 리소스가 현재 없으므로 새로 만들지 말고, 제공되기 전에는 코드 연결 지점만 설계한다.
- 자동 전투 회귀 테스트와 온라인 멀티플레이는 아직 구현되지 않았다. 다음 작업은 `task.md` 순서를 따른다.
