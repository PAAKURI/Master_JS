## P0 — 다음 작업 전에 수정할 결함

### GAME-001 라운드 종료 후 총알 판정 차단

- [ ] `EndRound` 진입 즉시 남은 총알을 제거하거나 전투 판정을 정지한다.
- [ ] 라운드가 `Playing` 상태가 아닐 때 `Player.TakeHit`가 체력과 속도를 변경하지 않게 한다.
- [ ] 승자 표시 3초 동안 HUD 체력과 승패가 추가 총알 때문에 바뀌지 않는지 확인한다.

근거: 현재 `_roundActive`는 사망 신호 처리만 막는다. 기존 총알은 계속 물리 처리되며 `TakeHit`를 호출할 수 있어 라운드 종료 화면 중 체력, 넉백, 사망 상태가 변경될 수 있다.

완료 조건:

1. 총알이 날아가는 중 한 플레이어가 사망해도 이후 총알은 누구에게도 피해를 주지 않는다.
2. 다음 라운드 시작 시 총알, 체력, 속도, 무적, 패링, 탄약 상태가 모두 초기값이다.

### GAME-002 동시 사망 판정 원자화

- [ ] 사망 신호가 들어오는 즉시 승자를 확정하지 말고 같은 물리 프레임의 사망 결과를 모아 한 번만 판정한다.
- [ ] 체력 동시 사망, 동시 낙사, 총알 피격과 낙사 동시 발생 테스트를 추가한다.
- [ ] 동시 사망 정책을 확정한다: 현재 UI 문구 기준으로는 무승부 및 라운드 재경기가 가장 일관적이다.

근거: `Kill()`의 신호는 즉시 처리된다. 첫 번째 플레이어의 `Died` 콜백 시 상대가 아직 살아 있으면 바로 승자가 결정되고 `_roundActive`가 꺼지므로, 현재 `둘 다 사망` 분기는 일반적인 동시 충돌 상황에서 사실상 도달하기 어렵다.

### GAME-003 라운드/씬 비동기 흐름 보호

- [ ] `StartMatch`, `EndRound`의 `async void` 흐름을 명시적 게임 상태 머신과 취소 가능한 전환 작업으로 교체한다.
- [ ] 카운트다운 중 ESC로 씬을 나갈 때 대기 중인 continuation이 해제된 노드/UI에 접근하지 않게 한다.
- [ ] 재시작 또는 씬 전환 시 기존 타이머가 이후 라운드를 중복 시작하지 않는지 확인한다.

근거: 현재 타이머 대기 작업은 씬 수명과 연결된 취소 수단이 없다. 씬 전환 뒤 비동기 continuation이 재개되면 해제된 노드 접근 또는 중복 상태 전환 위험이 있다.

- [ ] 항상 무시 정책이면 `ResolveCollision`/`TakeHit`에서 `OwnerPlayer == target`을 거부한다.
- [ ] 유예 정책이면 0.15초 뒤 포물선으로 돌아온 자기 총알이 자신을 맞힐 수 있음을 게임 규칙과 테스트에 명시한다.
- [ ] 패링 후에는 이전 소유자를 정상적으로 맞히고 새 소유자는 정해진 정책에 따라 보호되는지 확인한다.

- [ ] 기획 미정 사항인 숙인 상태 이동 가능 여부와 속도를 확정한다.

### COMBAT-002 패링 판정 프레임 검증
